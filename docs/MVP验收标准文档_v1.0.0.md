# MVP 验收标准文档 v1.0.1

## 1. 概述

### 1.1 文档目的

本文档定义旅游攻略生成平台 **MVP（最小可行产品）** 的验收标准，作为开发完成后的验收依据。

### 1.2 MVP 范围

| 模块       | MVP 包含功能                                             | MVP 不包含                |
| ---------- | -------------------------------------------------------- | ------------------------- |
| 攻略生成   | ✅ SSE 流式生成、渐进渲染                                | ❌ 实时交通/天气 API      |
| 行程展示   | ✅ 行程卡片、景点/餐饮/住宿卡片                          | ❌ 地图模式               |
| 大交通     | ✅ 静态经验推荐（高铁/普速火车P0/航班/大巴P1）+ 免责声明 | ❌ 实时车次/航班/大巴查询 |
| 修改建议   | ✅ 小程序端提出修改建议、原用户确认                      | ❌ 实时协同编辑           |
| 分享       | ✅ 生成 shareToken、查看分享攻略                         | ❌ 多人协同编辑           |
| 导出       | ✅ 导出为图片（小红书/朋友圈）                           | ❌ 导出 PDF/Word          |
| 城市知识库 | ✅ 静态 POI 数据、经验交通时间                           | ❌ 实时数据接入           |

### 1.3 验收等级定义

| 等级               | 说明                   | 通过标准   |
| ------------------ | ---------------------- | ---------- |
| **P0（必须通过）** | 核心功能，影响主流程   | 100% 通过  |
| **P1（应该通过）** | 重要功能，影响用户体验 | ≥ 90% 通过 |
| **P2（可以接受）** | 锦上添花功能           | ≥ 70% 通过 |

## 5. 验收流程

### 5.1 UAT（用户验收测试）流程

```
开发完成
   ↓
开发自测（开发团队内部）
   ↓
产品验收（产品经理验证核心流程）
   ↓
UAT 测试（真实用户试用）
   ↓
Bug 修复
   ↓
回归测试
   ↓
发布上线
```

### 5.2 Bug 分级与处理

| Bug 等级       | 说明                       | 处理时限      |
| -------------- | -------------------------- | ------------- |
| **P0（致命）** | 核心功能不可用，阻塞主流程 | 24 小时内修复 |
| **P1（严重）** | 重要功能异常，影响用户体验 | 3 天内修复    |
| **P2（一般）** | 轻微问题，不影响核心功能   | 下个版本修复  |
| **P3（建议）** | 优化建议，可延后处理       | 按需修复      |

### 5.3 验收通过标准

| 等级        | 通过标准                     |
| ----------- | ---------------------------- |
| **P0 功能** | 100% 通过，0 个 P0 Bug       |
| **P1 功能** | ≥ 90% 通过，≤ 2 个 P1 Bug    |
| **P2 功能** | ≥ 70% 通过，≤ 5 个 P2 Bug    |
| **非功能**  | 性能/兼容性/安全验收全部通过 |
