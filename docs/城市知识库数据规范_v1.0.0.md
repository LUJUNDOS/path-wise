# 城市知识库数据规范 v1.0.0

## 1. 概述

### 1.1 文档目的

本文档定义旅游攻略生成平台的**城市知识库数据规范**，包括：

- POI（兴趣点）字段标准
- 静态交通数据格式
- 数据来源标注规范

知识库是攻略生成的数据底座，数据质量直接决定攻略的准确性和可用性。

### 1.2 适用范围

| 使用者      | 用途                            |
| ----------- | ------------------------------- |
| 后端开发    | 数据库表设计、数据入库校验      |
| 数据采集    | 爬取/接入第三方数据时的字段映射 |
| AI 生成服务 | 读取知识库数据，生成攻略内容    |
| 前端开发    | 理解数据格式，正确展示          |

### 1.3 MVP 范围说明

| 数据类型     | MVP 阶段                                  | 阶段二             |
| ------------ | ----------------------------------------- | ------------------ |
| POI 基础数据 | ✅ 静态知识库（手动录入 + 高德 POI 导入） | 实时更新           |
| 开放时间     | ✅ 静态（每周更新）                       | 实时官方接口       |
| 票价信息     | ✅ 静态（每月更新）                       | 实时查询           |
| 市内交通路线 | ✅ 静态经验数据                           | 高德路线规划 API   |
| 大交通数据   | ✅ 静态经验推荐                           | 12306/航旅纵横 API |

### 1.4 与高德 API 的职责边界

> **设计原则**：高德给「数据」，知识库给 LLM「判断力」。两者不重叠，缺一不可。

#### 高德 API 负责的内容（知识库不重复存储）

| 问题                        | 高德能回答       | 理由                   |
| --------------------------- | ---------------- | ---------------------- |
| POI 的精确坐标              | ✅ 实时查询      | 高德数据权威且时效性强 |
| A → B 路线规划（距离/时间） | ✅ 实时计算      | 需考虑实时路况         |
| 周边 POI 搜索               | ✅ 实时搜索      | 动态结果，按距离排序   |
| 住宿/餐饮实时价格           | ✅ 通过 OTA 获取 | 每日变化，静态存无意义 |

#### 城市知识库负责的内容（高德给不了）

| 问题                               | 知识库能回答                | 存储字段                                 |
| ---------------------------------- | --------------------------- | ---------------------------------------- |
| 这个景点值不值得去？推荐游览多久？ | ✅ 人工精选判断             | `visitInfo.recommendedDuration` + `tips` |
| 从北京到长沙怎么去最合适？         | ✅ 静态经验推荐             | `intercity_transport.suggestions`        |
| 坡子街和太平老街先去哪个体验更好？ | ✅ curated 路线逻辑         | `tags` + 运营标注                        |
| 景点国庆闭馆吗？                   | ✅ 人工维护特殊安排         | `specialSchedule`                        |
| 必吃的 3 道湘菜是什么？            | ✅ 运营精选，不是评分高就好 | `menuInfo.mustTry`                       |
| 排队需要多久？有什么坑？           | ✅ 经验数据                 | `queueInfo` + `tips`                     |

#### 精简原则（MVP 阶段执行）

- **不存**：住宿实时价格（`roomInfo.priceRange` 仅记录参考区间，不做精确展示）
- **不存**：POI 坐标（依赖高德 API 动态获取，知识库只存少量地标用于 LLM 描述）
- **不存**：市内导航路线详情（高德 API 实时规划）
- **重点存**：经验类、判断类、精选类内容——这是高德无法提供的差异化价值

## 2. POI 字段标准

### 2.1 基础字段（所有 POI 必填）

| 字段名        | 类型          | 必填 | 说明                                          | 示例                             |
| ------------- | ------------- | ---- | --------------------------------------------- | -------------------------------- |
| `poiId`       | String        | ✅   | 全局唯一 ID，格式：`{城市拼音}_{分类}_{序号}` | `changsha_attraction_001`        |
| `poiName`     | String        | ✅   | POI 名称（中文）                              | `岳麓山风景名胜区`               |
| `poiNameEn`   | String        | ❌   | POI 英文名称（国际化用）                      | `Yuelu Mountain`                 |
| `category`    | String        | ✅   | POI 分类（见 2.2 节）                         | `attraction`                     |
| `subCategory` | String        | ❌   | 子分类                                        | `mountain`                       |
| `latitude`    | Number        | ✅   | 纬度（GCJ-02 坐标系）                         | `28.226`                         |
| `longitude`   | Number        | ✅   | 经度（GCJ-02 坐标系）                         | `112.938`                        |
| `address`     | String        | ✅   | 结构化地址                                    | `湖南省长沙市岳麓区麓山南路`     |
| `cityCode`    | String        | ✅   | 所属城市编码（GB/T 2260）                     | `430100`（长沙）                 |
| `cityName`    | String        | ✅   | 所属城市名称                                  | `长沙`                           |
| `description` | String        | ❌   | 简介（≤ 200 字）                              | `岳麓山是长沙市的标志性景点...`  |
| `tags`        | Array<String> | ❌   | 标签（见 2.3 节）                             | `["免费", "适合拍照", "需预约"]` |
| `coverImage`  | String        | ❌   | 封面图 URL（宽高比 16:9）                     | `https://example.com/xxx.jpg`    |
| `source`      | String        | ✅   | 数据来源（见第 4 节）                         | `gaode_poi`                      |
| `sourceUrl`   | String        | ❌   | 数据来源 URL                                  | `https://www.amap.com/...`       |
| `updateTime`  | String        | ✅   | 数据最后更新时间（ISO 8601）                  | `2026-06-20T00:00:00+08:00`      |

### 2.2 分类体系（category）

采用**三级分类体系**：一级分类（必填）→ 二级分类（必填）→ 三级标签（可选）。

#### 一级分类（8 类）

| 分类代码        | 分类名称  | 说明                           |
| --------------- | --------- | ------------------------------ |
| `attraction`    | 景点/景区 | 自然风光、人文古迹、主题乐园等 |
| `restaurant`    | 餐饮      | 正餐、小吃、咖啡厅、酒吧等     |
| `accommodation` | 住宿      | 酒店、民宿、青旅等             |
| `transport`     | 交通枢纽  | 高铁站、机场、长途汽车站等     |
| `shopping`      | 购物      | 商场、特色街区、免税店等       |
| `entertainment` | 娱乐      | 演出、夜生活、SPA 等           |
| `civic`         | 公共设施  | 医院、派出所、游客中心等       |
| `other`         | 其他      | 不满足以上分类的 POI           |

#### 二级分类（示例）

**景点（attraction）**：

- `scenic`：风景名胜区（如岳麓山）
- `historical`：历史文化遗址（如岳麓书院）
- `park`：城市公园（如橘子洲）
- `theme_park`：主题乐园（如长沙世界之窗）
- `religious`：宗教场所（如开福寺）
- `neighborhood`：特色街区（如太平老街）

**餐饮（restaurant）**：

- `local`：本地菜（如湘菜馆）
- `snack`：小吃/夜市（如坡子街）
- `cafe`：咖啡厅
- `bar`：酒吧/清吧
- `street_food`：路边摊/美食街

**住宿（accommodation）**：

- `hotel`：酒店
- `guesthouse`：民宿/客栈
- `hostel`：青年旅舍
- `apartment`：短租公寓

#### 三级标签（自由标签）

由运营人员根据 POI 特点手动添加，用于精细筛选。

示例：

- 景点：`["适合拍照", "适合亲子", "需预约", "免费", "世界遗产"]`
- 餐饮：`["辣", "湘菜", "排队久", "可外卖", "有包厢"]`
- 住宿：`["性价比高", "近地铁", "含早餐", "有泳池"]`

### 2.3 标签体系（tags）

标签用于**快速筛选**和**冲突检测**，分为系统标签和用户标签。

#### 系统标签（ reserved\_ 前缀）

| 标签                     | 适用分类                  | 说明                   |
| ------------------------ | ------------------------- | ---------------------- |
| `reserved_free`          | attraction                | 免费开放               |
| `reserved_need_booking`  | attraction, restaurant    | 需要预约               |
| `reserved_seasonal`      | attraction                | 季节性开放（如樱花季） |
| `reserved_closed_monday` | attraction                | 周一闭馆               |
| `reserved_spicy`         | restaurant                | 辣（湘菜/川菜等）      |
| `reserved_vegetarian`    | restaurant                | 有素食选项             |
| `reserved_near_subway`   | accommodation, attraction | 近地铁站（≤ 500m）     |
| `reserved_airport_hotel` | accommodation             | 机场/高铁站附近酒店    |

#### 用户标签（无前缀）

自由标签，由运营人员或用户评价中提取。

### 2.4 坐标与地址规范

#### 坐标系

**强制要求**：所有坐标必须使用 **GCJ-02 坐标系**（火星坐标系），与高德地图一致。

> ⚠️ **禁止**使用 WGS-84（GPS 原始坐标）或 BD-09（百度坐标），除非在入库时转换。

#### 地址结构化规范

地址字段 `address` 必须遵循以下格式：

```
{省}{市}{区/县}{街道/镇}{详细地址}
```

示例：

- ✅ `湖南省长沙市岳麓区麓山南路`
- ✅ `湖南省长沙市芙蓉区解放西路`
- ❌ `岳麓区麓山南路`（缺少省、市）
- ❌ `长沙岳麓山`（格式不标准）

## 3. 景点数据扩展字段

仅适用于 `category=attraction` 的 POI。

### 3.1 开放时间（openingHours）

采用 **OSM opening_hours 规范**（简化版），支持周期性时间和特殊安排。

#### 格式规范

```
# 周期性开放时间
"openingHours": [
  {
    "period": "周一-周日",          # 或 "周一-周五", "周六-周日"
    "hours": "08:00-18:00",       # 24 小时制
    "note": "最晚入园 17:00"       # 可选
  }
]

# 特殊安排（节假日/临时闭馆）
"specialSchedule": [
  {
    "date": "2026-10-01",         # 具体日期（ISO 8601）
    "status": "closed",           # closed / open / limited
    "hours": "",                  # status=open 时填写
    "note": "国庆节闭馆"
  }
]
```

#### 示例

```json
{
  "poiId": "changsha_attraction_001",
  "poiName": "岳麓书院",
  "openingHours": [
    {
      "period": "周一-周日",
      "hours": "08:00-17:30",
      "note": "最晚入园 17:00"
    }
  ],
  "specialSchedule": [
    {
      "date": "2026-10-01",
      "status": "closed",
      "note": "国庆节闭馆"
    }
  ]
}
```

### 3.2 票价信息（ticketInfo）

#### 格式规范

```json
{
  "isFree": false, // 是否免费
  "priceRange": "¥40/人", // 价格描述（展示用）
  "prices": [
    // 详细票价（计算用）
    {
      "type": "成人票",
      "price": 40,
      "currency": "CNY"
    },
    {
      "type": "学生票",
      "price": 20,
      "currency": "CNY",
      "condition": "凭学生证"
    }
  ],
  "discount": [
    // 优惠政策
    {
      "type": "免票",
      "condition": "14 岁以下儿童、70 岁以上老人"
    }
  ],
  "bookingPlatform": [
    // 可预约平台
    "美团",
    "携程",
    "官方公众号"
  ]
}
```

### 3.3 游览信息（visitInfo）

```json
{
  "recommendedDuration": 120, // 推荐游览时长（分钟）
  "bestSeason": ["春季", "秋季"], // 最佳游览季节
  "bestTimeOfDay": ["上午", "傍晚"], // 一天中最佳时段
  "crowdLevel": "high", // 拥挤程度：low / medium / high
  "crowdPeakTime": "10:00-15:00", // 高峰时段
  "tips": [
    // 游玩建议
    "建议从南门入园，步行上山约 30 分钟",
    "山顶可俯瞰长沙全景，适合拍照"
  ]
}
```

## 4. 餐饮数据扩展字段

仅适用于 `category=restaurant` 的 POI。

### 4.1 营业时间（businessHours）

格式与景点开放时间类似，但支持分段营业。

```json
{
  "businessHours": [
    {
      "period": "周一-周日",
      "hours": "11:00-14:00, 17:00-21:00", // 午市 + 晚市
      "note": "最后点单 20:30"
    }
  ]
}
```

### 4.2 菜品与价格（menuInfo）

```json
{
  "averagePrice": 80, // 人均价格（元）
  "priceRange": "¥50-100/人", // 价格区间（展示用）
  "mustTry": [
    // 必点菜品
    {
      "dishName": "剁椒鱼头",
      "price": 68,
      "description": "湘菜经典，辣度 ★★★★"
    }
  ],
  "dietaryOptions": [
    // 饮食选项
    "辣",
    "可微辣",
    "有素食"
  ]
}
```

### 4.3 排队信息（queueInfo）

```json
{
  "waitTimeTypical": 30, // 典型排队时间（分钟）
  "waitTimePeak": 60, // 高峰排队时间（分钟）
  "peakHours": ["12:00-13:00", "18:30-19:30"],
  "reservation": true, // 是否支持预约
  "reservationPlatform": ["大众点评", "美团"]
}
```

## 5. 住宿数据扩展字段

仅适用于 `category=accommodation` 的 POI。

### 5.1 房型与价格（roomInfo）

> ⚠️ **MVP 精简说明**：住宿价格每日变动，不适合在知识库中维护精确价格。
> `priceRange` 字段仅存储**参考区间**（如 `¥200-600/晚`），用于 LLM 生成价格描述语句；
> 实际订房价格应引导用户前往携程/美团等平台实时查询。
> 阶段二接入 OTA API 后，此字段可废弃。

```json
{
  "roomTypes": [
    {
      "type": "标准间",
      "priceRange": "¥300-500/晚",
      "amenities": ["独立卫浴", "免费WiFi", "空调"],
      "maxGuests": 2
    },
    {
      "type": "家庭房",
      "priceRange": "¥500-800/晚",
      "amenities": ["独立卫浴", "免费WiFi", "空调", "儿童床"],
      "maxGuests": 4
    }
  ],
  "breakfast": true, // 是否含早餐
  "breakfastPrice": 58, // 早餐价格（元/人），不含早餐时为 null
  "cancellationPolicy": "入住前 1 天 18:00 前免费取消"
}
```

### 5.2 位置与交通（locationInfo）

```json
{
  "distanceToCenter": 5, // 到市中心距离（公里）
  "nearbySubway": [
    // 附近地铁站
    {
      "stationName": "五一广场站",
      "distance": 500, // 步行距离（米）
      "walkTime": 7 // 步行时间（分钟）
    }
  ],
  "nearbyAttractions": [
    // 附近景点（3 公里内）
    {
      "poiId": "changsha_attraction_001",
      "poiName": "岳麓山",
      "distance": 2000
    }
  ]
}
```

## 6. 静态交通数据格式

### 6.1 城市交通枢纽（transport_hubs）

每个城市维护一个交通枢纽表，用于大交通推荐。

```json
{
  "cityCode": "430100",
  "cityName": "长沙",
  "hubs": [
    {
      "hubId": "changsha_hub_001",
      "hubName": "长沙南站",
      "hubType": "high_speed_rail", // high_speed_rail / airport / long_distance_bus
      "latitude": 28.154,
      "longitude": 113.019,
      "address": "湖南省长沙市雨花区花侯路",
      "toCityCenter": {
        // 到市中心交通
        "subway": {
          "line": "地铁2号线",
          "duration": 45, // 分钟
          "price": 5, // 元
          "note": "直达五一广场"
        },
        "taxi": {
          "duration": 40,
          "priceRange": "50-70",
          "note": "高峰期可能堵车"
        },
        "airport_shuttle": null // 机场大巴（机场专用）
      }
    },
    {
      "hubId": "changsha_hub_002",
      "hubName": "长沙黄花国际机场",
      "hubType": "airport",
      "latitude": 28.193,
      "longitude": 113.22,
      "address": "湖南省长沙市长沙县空港城",
      "toCityCenter": {
        "subway": {
          "line": "地铁6号线",
          "duration": 60,
          "price": 8,
          "note": "直达迎宾路口站"
        },
        "taxi": {
          "duration": 50,
          "priceRange": "100-130"
        },
        "airport_shuttle": {
          "duration": 50,
          "price": 20,
          "frequency": "15分钟/班"
        }
      }
    }
  ]
}
```

### 6.2 市内交通经验数据（city_transport）

每个城市维护一个市内交通经验数据表，用于攻略生成时的交通时间估算。

```json
{
  "cityCode": "430100",
  "cityName": "长沙",
  "transportModes": [
    {
      "mode": "subway", // 交通方式：subway / bus / taxi / walk
      "description": "地铁",
      "coverage": "较好", // 覆盖程度：good / medium / poor
      "pricePerKm": null, // 按里程计价时为每公里价格
      "startingPrice": 2, // 起步价（元）
      "note": "目前开通 6 条线路，基本覆盖主要景点"
    },
    {
      "mode": "taxi",
      "description": "打车",
      "startingPrice": 8,
      "pricePerKm": 2.4,
      "note": "高峰期难打车，建议使用网约车"
    },
    {
      "mode": "shared_bike",
      "description": "共享单车",
      "startingPrice": 1.5,
      "note": "适合短距离骑行（≤ 3 公里）"
    }
  ],
  "typicalRouteDuration": [
    // 典型路线耗时（经验数据）
    {
      "from": "五一广场",
      "to": "岳麓山南门",
      "mode": "subway",
      "duration": 40,
      "note": "地铁2号线 → 溁湾镇站，步行 10 分钟"
    },
    {
      "from": "五一广场",
      "to": "长沙南站",
      "mode": "subway",
      "duration": 45,
      "note": "地铁2号线直达"
    }
  ]
}
```

### 6.3 跨城交通静态数据（intercity_transport）

MVP 阶段不调实时 API，使用**静态经验数据**推荐大交通方案。

- `high_speed_rail`：高铁/动车（P0）

## 8. 数据入库校验规则

### 8.1 必填字段校验

| 字段                     | 校验规则           | 错误提示                    |
| ------------------------ | ------------------ | --------------------------- |
| `poiId`                  | 全局唯一           | `poiId already exists`      |
| `poiName`                | 非空，≤ 100 字符   | `poiName is required`       |
| `latitude` / `longitude` | 有效坐标范围       | `invalid coordinates`       |
| `category`               | 必须在分类体系中   | `invalid category`          |
| `source`                 | 必须在来源代码中   | `invalid source`            |
| `updateTime`             | 有效 ISO 8601 格式 | `invalid updateTime format` |

### 8.2 坐标校验

```python
def validate_coordinates(lat, lng):
    """校验坐标是否在中华人民共和国境内（含南海）"""
    if not (3.0 <= lat <= 53.0):
        return False, "latitude out of range"
    if not (73.0 <= lng <= 135.0):
        return False, "longitude out of range"
    return True, "ok"
```

### 8.3 地址校验

```python
def validate_address(address):
    """校验地址是否包含省、市信息"""
    if not address:
        return False, "address is required"
    # 简单校验：至少包含 2 个层级（省 + 市）
    if address.count("省") < 1 or address.count("市") < 1:
        return False, "address must include province and city"
    return True, "ok"
```

### 9.4 知识库维护工作流（Obsidian + Git → JSON）

#### 背景

城市知识库是「经验类、判断类、精选类」数据，由运营人员手工维护，不适合直接在数据库界面操作。推荐采用 **Obsidian 编辑 → Git 版本管理 → 自动校验 → JSON 导入** 的分层工作流。

#### 工作流架构

```
┌─────────────────────┐       ┌───────────────────────┐       ┌────────────────────┐
│  Obsidian（编辑层）  │──push─▶  Git 仓库（版本管理）  │──CI──▶   JSON 文件（数据层）│
│                     │       │                       │       │                    │
│ • 人工编写内容       │       │ • 变更追踪             │       │ • 结构化，程序可读  │
│ • YAML frontmatter  │       │ • PR Review           │       │ • LLM 可直接消费   │
│ • Markdown 正文      │       │ • 历史回溯             │       │ • Schema 校验通过  │
└─────────────────────┘       └───────────────────────┘       └────────────────────┘
       运营编辑                      协作管控                        系统消费
```

#### 具体步骤

**Step 1：Obsidian 中编写 POI**

```python
# scripts/validate.py 核心校验逻辑
def validate_poi(poi_data: dict) -> list[str]:
    errors = []
    required_fields = ['poiId', 'poiName', 'category', 'cityCode', 'source', 'updateTime']
    for field in required_fields:
        if field not in poi_data:
            errors.append(f"缺少必填字段: {field}")
    if 'latitude' in poi_data:
        if not (3.0 <= poi_data['latitude'] <= 53.0):
            errors.append("纬度超出中国范围")
    if 'category' in poi_data:
        valid_categories = ['attraction', 'restaurant', 'accommodation', 'transport', 'shopping', 'entertainment', 'civic', 'other']
        if poi_data['category'] not in valid_categories:
            errors.append(f"无效分类: {poi_data['category']}")
    return errors
```

#### LLM Wiki 适用场景

Obsidian 的「LLM Wiki」工作模式可用于以下场景：

| 场景                | 适合 | 示例                                            |
| ------------------- | ---- | ----------------------------------------------- |
| 城市文化背景撰写    | ✅   | 请 LLM 基于公开资料起草「长沙城市简介」Markdown |
| 批量初始化 POI 骨架 | ✅   | 请 LLM 生成 10 个景点的 YAML frontmatter 模板   |
| 校对已有内容        | ✅   | 请 LLM 检查某 POI 的 tips 是否准确              |
| 直接作为数据库      | ❌   | Obsidian 格式无 Schema 约束，不可直接被程序读取 |

#### 适用原则

- Obsidian 是**编辑界面**（人写）
- Git 是**协作和版本管理**层
- JSON 是**数据格式**（程序读）
- 三者分层，互不越权
