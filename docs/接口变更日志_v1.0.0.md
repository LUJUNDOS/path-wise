### API v1.0.6 —— 2026-06-20（当前版本）

**本次变更摘要**：大交通类型扩展，新增 `normal_train`（普速火车）和 `bus`（长途大巴）

#### 变更 1：POST /trips/generate —— 大交通类型扩展

**变更类型**：🟢 **New**

**变更前**：

```typescript
interface DestinationConfig {
  transportTo: 'high_speed_rail' | 'flight' | 'auto';
  transportBack?: 'high_speed_rail' | 'flight' | 'auto';
}
```

**变更后**：

```typescript
interface DestinationConfig {
  transportTo: 'high_speed_rail' | 'normal_train' | 'flight' | 'bus' | 'auto';
  transportBack?: 'high_speed_rail' | 'normal_train' | 'flight' | 'bus' | 'auto';
}
```

**影响范围**：

- 前端：行程配置页面的大交通选择器，需新增 `normal_train` 和 `bus` 选项
- 后端：Trip Lifecycle 引擎，需处理新增交通类型

**缓冲时间规则更新**：

```typescript
// 新增规则
function getBufferMinutes(transportType: TransportType): number {
  switch (transportType) {
    case 'flight':
      return 120; // 提前 2 小时（不变）
    case 'normal_train':
      return 90; // 提前 1.5 小时（新增）
    case 'high_speed_rail':
      return 90; // 提前 1.5 小时（不变）
    case 'bus':
      return 60; // 提前 1 小时（新增）
    default:
      return 90;
  }
}
```

**兼容性说明**：

- 旧请求（仅含 `high_speed_rail` / `flight` / `auto`）继续有效
- 新增类型对旧版前端不可见，需前端同步更新

#### 变更 2：POST /transport/search —— 响应结构新增字段

**变更类型**：🟢 **New**

**新增响应字段**：

| 字段                    | 类型      | 说明                | 适用交通类型          |
| ----------------------- | --------- | ------------------- | --------------------- |
| `isOvernight`           | `boolean` | 是否为隔夜车次/班次 | `normal_train`, `bus` |
| `pricePerPerson.硬座`   | `number`  | 硬座价格            | `normal_train`        |
| `pricePerPerson.硬卧`   | `number`  | 硬卧价格            | `normal_train`        |
| `pricePerPerson.软卧`   | `number`  | 软卧价格            | `normal_train`        |
| `pricePerPerson.普通座` | `number`  | 大巴普通座价格      | `bus`                 |
| `pricePerPerson.商务座` | `number`  | 大巴商务座价格      | `bus`                 |

**新增 type 枚举值**：

```typescript
type TransportType =
  | 'high_speed_rail' // 已有
  | 'flight' // 已有
  | 'normal_train' // 新增 v1.0.6
  | 'bus'; // 新增 v1.0.6
```

**兼容性说明**：

- 旧前端代码访问新字段会得到 `undefined`，需更新前端以显示新字段
- `arriveTime` 字段对隔夜车次返回「次日 HH:MM」格式字符串

## Breaking Change 处理规范

当出现 🔴 Breaking Change 时，遵循以下流程：

### 1. 通知机制

- 提前至少 2 个版本宣告（先 Deprecated，再 Breaking）
- 在 API 响应 Header 中添加 `Deprecation` 标识

### 2. 过渡期策略

```
旧版本 → [2 个版本过渡期] → Breaking Change 生效
旧接口设置 Deprecated 标识 → 旧接口彻底移除
```

### 3. 迁移指引格式

每次 Breaking Change 必须提供迁移示例：

```typescript
// 迁移前（v1.x）
const body = { transportTo: 'random' };

// 迁移后（v2.x）
const body = { transportTo: 'auto' };
// 说明：'random' 已重命名为 'auto'
```

## 接口版本命名规则

```
API vMAJOR.MINOR.PATCH
      │      │      └── 修复/文档更新（不改变接口行为）
      │      └── 新增接口/字段（向下兼容）
      └── 破坏性变更（不兼容）
```

**示例**：

- `v1.0.6` → `v1.0.7`：修复某接口返回值计算错误
- `v1.0.6` → `v1.1.0`：新增 `/auth/register` 接口
- `v1.x.x` → `v2.0.0`：`POST /trips/generate` 结构重设计
