## 1. 设计原则

### 1.1 核心原则

| 原则             | 说明                           | PATH-WISE 应用                              |
| ---------------- | ------------------------------ | ------------------------------------------- |
| **反馈及时**     | 用户每次操作都要有即时反馈     | 按钮点击 → 立即 Loading；SSE 进度条实时更新 |
| **容错友好**     | 错误要清晰可恢复               | 提供重试按钮；保留已填写的表单内容          |
| **渐进呈现**     | 先展示核心内容，再逐步加载详情 | SSE 边生成边展示；图片懒加载                |
| **降低认知负担** | 每个页面聚焦一个核心任务       | 城市选择页只做选择；配置页只做配置          |
| **感知性能**     | 即使加载慢，也要让用户感觉快   | Skeleton 占位；流式展示生成结果             |

### 1.2 视觉设计原则

```
色彩系统（基于 shadcn/ui + Tailwind CSS）

主色调：深海蓝 #1D4ED8（行动按钮、链接、高亮）
辅助色：橙金 #F59E0B（重要提示、标签）
成功色：翠绿 #10B981（成功状态）
警告色：橙色 #F97316（免责声明、注意事项）
错误色：红色 #EF4444（错误提示）
中性色：灰阶 #F9FAFB → #111827

字体：
  中文：Noto Sans SC（Google Fonts）
  数字：Tabular Numbers（等宽数字，对齐时间显示）
  代码：JetBrains Mono

圆角：
  小组件（Tag/Badge）：rounded-full（完全圆角）
  卡片：rounded-2xl（16px）
  按钮：rounded-xl（12px）
  输入框：rounded-lg（8px）

阴影：
  卡片悬浮：shadow-md（默认状态 shadow-sm）
  弹窗：shadow-2xl
```

## 2. 加载状态设计

### 2.1 加载状态类型

| 类型                       | 适用场景               | 组件               | 最小显示时间      |
| -------------------------- | ---------------------- | ------------------ | ----------------- |
| **Skeleton（骨架屏）**     | 内容加载（列表、卡片） | `<Skeleton>`       | -                 |
| **Spinner（旋转图标）**    | 按钮 Loading、小区域   | `<Spinner>`        | 200ms（避免闪烁） |
| **Progress Bar（进度条）** | 文件上传、攻略生成     | `<Progress>`       | -                 |
| **全屏蒙层**               | 全页面操作（极少使用） | `<LoadingOverlay>` | 禁止超过 5s       |

```tsx
          />
        </div>

        {/* 进度文字 */}
        <div className="flex justify-between text-xs text-gray-400">
          <span>{getStageLabel(stage)}</span>
          <span>{progress}%</span>

      </div>

      {/* 已生成的天（边生成边展示） */}
      {generatedDays.length > 0 && (
        <div className="space-y-3">
          <p className="text-sm font-medium text-gray-600">已生成的行程：</p>

            <DayPlanCard key={day.dayIndex} day={day} isPreview />
          ))}
        </div>
      )}

      {/* 取消按钮 */}
      <button
        className="w-full py-3 text-sm text-gray-400 hover:text-gray-600 transition-colors"
        onClick={onCancel}
      >
        取消生成
      </button>
    </div>
  );
}

// 阶段标签映射
function getStageLabel(stage: SSEProgressStage): string {
  const labels: Record<SSEProgressStage, string> = {
    initializing: "正在分析你的需求...",
    fetching_data: "正在获取目的地数据...",
    generating: "正在规划行程路线...",
    optimizing: "正在优化时间安排...",
    finalizing: "正在完成最后细节...",
  };
  return labels[stage];
}
```

## 3. 动画与过渡效果

### 3.1 动画原则

```
原则 1：目的性动画
  - 每个动画都有明确目的（引导注意力、反馈状态、建立层级关系）
  - 禁止纯装饰性动画（用户会感到被打扰）

原则 2：时长规范
  - 微交互（按钮状态切换）：100-150ms
  - 页面内元素进入：200-300ms
  - 页面切换：250-400ms
  - 复杂动画（如展开/收起）：300-500ms
  - 超过 500ms 的动画需要有充分理由

原则 3：缓动函数
  - 进入：ease-out（由快到慢，自然落地感）
  - 退出：ease-in（由慢到快，自然离开感）
  - 状态切换：ease-in-out（对称）
```

### 3.3 页面切换动画

```tsx
// 使用 Framer Motion 实现页面切换动画
import { motion, AnimatePresence } from "framer-motion";
import { useLocation } from "react-router-dom";

const pageVariants = {
  initial: {
    opacity: 0,
    y: 12,
  },
  animate: {
    opacity: 1,
    y: 0,
    transition: {
      duration: 0.25,
      ease: "easeOut",
    },
  },
  exit: {
    opacity: 0,
    y: -12,
    transition: {
      duration: 0.2,
      ease: "easeIn",
    },
  },
};

function PageWrapper({ children }: { children: React.ReactNode }) {
  const location = useLocation();

  return (
    <AnimatePresence mode="wait">
      <motion.div
        key={location.pathname}
        variants={pageVariants}
        initial="initial"
        animate="animate"
        exit="exit"
      >
        {children}
      </motion.div>
    </AnimatePresence>
  );
}
```

## 4. 错误状态设计

### 4.1 错误类型与展示方式

| 错误类型         | 展示方式                   | 示例                   |
| ---------------- | -------------------------- | ---------------------- |
| **表单验证错误** | 字段下方红色提示文字       | "日期格式不正确"       |
| **网络请求失败** | Toast 提示（可关闭）       | "网络异常，请检查连接" |
| **权限不足**     | 全屏错误页（含返回按钮）   | "请先登录后再查看"     |
| **资源不存在**   | 404 页面                   | "该行程不存在"         |
| **攻略生成失败** | 内联错误卡片（含重试按钮） | "生成失败，点击重试"   |
| **服务器错误**   | 全屏错误页（含重试按钮）   | "服务异常，请稍后重试" |

### 4.2 Toast 提示规范

```tsx

// 警告提示

// Toast 位置：右下角（不遮挡内容区）
// 移动端：底部居中（更符合拇指操作区域）
```

## 7. 无障碍设计规范（a11y）

### 7.1 WCAG 2.1 AA 目标

PATH-WISE 目标达到 WCAG 2.1 AA 级别，核心要求：

| 类别       | 要求                   | 实现方式                                   |
| ---------- | ---------------------- | ------------------------------------------ |
| **可感知** | 图片有替代文字         | `<img alt="北京天安门广场">`               |
|            | 颜色不是唯一区分手段   | 错误状态同时用颜色 + 图标                  |
|            | 文字对比度 ≥ 4.5:1     | Tailwind 默认颜色满足                      |
| **可操作** | 键盘可完全操作         | 确保所有交互可键盘访问                     |
|            | 跳过重复内容           | 添加 Skip Navigation 链接                  |
|            | 无时间限制（或可延长） | 避免强制自动跳转                           |
| **可理解** | 语言声明               | `<html lang="zh-CN">`                      |
|            | 错误提示明确           | "请输入有效的出发日期（格式：YYYY-MM-DD）" |
| **健壮性** | HTML 语义化            | 使用正确的语义元素                         |
|            | ARIA 属性正确          | 动态内容使用 `aria-live`                   |

### 7.2 语义化 HTML 规范

```tsx

      className="p-2 rounded-full hover:bg-gray-100"

    >

  );
```
