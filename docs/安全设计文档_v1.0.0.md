## 1. 概述

### 1.1 文档目的

本文档定义 PATH-WISE 旅游攻略生成平台的安全设计，包括：

- 认证与授权机制
- 数据加密策略
- 输入验证与输出编码
- 安全头配置
- 日志与审计
- 安全测试策略

### 1.2 安全设计原则

| 原则             | 说明                                           |
| ---------------- | ---------------------------------------------- |
| **零信任架构**   | 不信任任何请求（即使来自内网）                 |
| **最小权限原则** | 只授予完成任务所需的最小权限                   |
| **纵深防御**     | 多层安全防护（WAF → API 网关 → 应用 → 数据库） |
| **安全左移**     | 在开发阶段就考虑安全（而非上线前才补救）       |
| **数据最小化**   | 只收集和存储必要的数据                         |

## 2. 安全威胁分析

### 2.1 常见威胁（OWASP Top 10）

| 威胁                      | 风险等级 | 防护措施                            |
| ------------------------- | -------- | ----------------------------------- |
| **注入攻击（SQL/NoSQL）** | 🔴 高    | 使用参数化查询（Prisma ORM）        |
| **身份认证失效**          | 🔴 高    | 使用 JWT + bcrypt，强制 HTTPS       |
| **敏感数据泄露**          | 🔴 高    | 数据加密（传输层 + 存储层）         |
| **XML 外部实体（XXE）**   | 🟡 中    | 禁用 XML 解析器（本项目不使用 XML） |
| **失效的访问控制**        | 🔴 高    | 基于角色的访问控制（RBAC）          |
| **安全配置错误**          | 🟡 中    | 使用环境变量，禁用调试模式          |
| **跨站脚本（XSS）**       | 🔴 高    | 输出编码（React 自动转义）          |
| **不安全的反序列化**      | 🟡 中    | 使用 JSON，避免反序列化不可信数据   |
| **使用含已知漏洞的组件**  | 🟡 中    | 定期更新依赖（npm audit）           |
| **日志记录和监控不足**    | 🟡 中    | 记录安全事件，配置告警              |

### 2.2 威胁建模（STRIDE）

```
┌─────────────────────────────────────────────────────┐
│                   STRIDE 威胁模型                    │
├─────────────────────────────────────────────────────┤
│  S - Spoofing（伪装）                             │
│    风险：攻击者伪装成合法用户                       │

│  E - Elevation of Privilege（提权）                │
│    风险：普通用户执行管理员操作                     │

└─────────────────────────────────────────────────────┘
```

## 3. 认证设计

### 3.1 认证方案选型

| 方案                      | 优势                  | 劣势                                            | 推荐场景              |
| ------------------------- | --------------------- | ----------------------------------------------- | --------------------- |
| **Session + Cookie**      | 简单，服务端可控制    | 扩展性差（多服务器需要共享 Session）            | 单体应用              |
| **JWT（JSON Web Token）** | 无状态，扩展性好      | Token 无法主动撤销（需要短过期时间 + 刷新机制） | ✅ 推荐（本文档采用） |
| **OAuth 2.0**             | 第三方登录（微信/QQ） | 实现复杂                                        | 可选（后续扩展）      |

### 3.2 JWT 认证设计

#### 3.2.1 Token 结构

```json
// Header
{
  "alg": "HS256",
  "typ": "JWT"
}

// Payload（不要存储敏感信息！）
{
  "sub": "user_001",           // 用户 ID
  "email": "user@example.com",  // 邮箱（非敏感）
  "role": "user",               // 角色
  "iat": 1700000000,           // 签发时间（Unix 时间戳）
  "exp": 1700003600,           // 过期时间（1 小时后）
  "jti": "unique-token-id"     // JWT ID（用于黑名单）
}

// Signature
// HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)
```

#### 3.2.2 双 Token 机制（Access Token + Refresh Token）

```
┌─────────────────────────────────────────────────────┐
│                  双 Token 认证流程                   │
├─────────────────────────────────────────────────────┤
│  1. 用户登录                                       │
│     → 服务器返回 Access Token（有效期 1 小时）      │
│     → 服务器返回 Refresh Token（有效期 7 天）       │
│     → Refresh Token 存储在 HttpOnly Cookie          │
├─────────────────────────────────────────────────────┤
│  2. 用户访问 API                                   │
│     → 请求头携带 Access Token：                    │
│       Authorization: Bearer <access_token>          │
│     → 服务器验证 Access Token                      │
│       - 有效：返回数据                             │
│       - 过期：返回 401 Unauthorized                │
├─────────────────────────────────────────────────────┤
│  3. Access Token 过期                              │
│     → 前端使用 Refresh Token 请求新 Access Token    │
│     → 服务器验证 Refresh Token：                    │
│       - 有效：返回新 Access Token                  │
│       - 过期/黑名单：返回 401，要求重新登录        │
├─────────────────────────────────────────────────────┤
│  4. 用户登出                                       │
│     → 前端删除 Access Token                        │
│     → 前端请求服务器撤销 Refresh Token（加入黑名单）│
└─────────────────────────────────────────────────────┘
```

#### 3.2.3 实现示例

```typescript
// apps/api/src/services/auth.service.ts
import jwt from 'jsonwebtoken';
import bcrypt from 'bcrypt';
import Redis from 'ioredis';

const redis = new Redis(process.env.REDIS_URL);
const JMT_SECRET = process.env.JWT_SECRET!;
const JMT_EXPIRES_IN = '1h';
const REFRESH_TOKEN_EXPIRES_IN = 7 * 24 * 3600; // 7 天（秒）

// 登录
export async function login(email: string, password: string) {
  // 1. 验证邮箱和密码
  const user = await prisma.user.findUnique({ where: { email } });
  if (!user || !(await bcrypt.compare(password, user.passwordHash))) {
    throw new Error('Invalid credentials');
  }

  // 2. 生成 Access Token
  const accessToken = jwt.sign(
    {
      sub: user.id,
      email: user.email,
      role: user.role,
    },
    JMT_SECRET,
    { expiresIn: JMT_EXPIRES_IN },
  );

  // 3. 生成 Refresh Token
  const refreshToken = jwt.sign({ sub: user.id }, JMT_SECRET, {
    expiresIn: REFRESH_TOKEN_EXPIRES_IN,
  });

  // 4. 将 Refresh Token 存储到 Redis（用于撤销）
  await redis.setex(`refresh_token:${user.id}:${refreshToken}`, REFRESH_TOKEN_EXPIRES_IN, 'valid');

  return { accessToken, refreshToken };
}

// 验证 Access Token
export function verifyAccessToken(token: string) {
  try {
    const payload = jwt.verify(token, JMT_SECRET);
    return payload;
  } catch (error) {
    throw new Error('Invalid or expired token');
  }
}

// 刷新 Access Token
export async function refreshAccessToken(refreshToken: string) {
  // 1. 验证 Refresh Token
  let payload: any;
  try {
    payload = jwt.verify(refreshToken, JMT_SECRET);
  } catch (error) {
    throw new Error('Invalid or expired refresh token');
  }

  // 2. 检查 Redis 中是否存在（未被撤销）
  const exists = await redis.exists(`refresh_token:${payload.sub}:${refreshToken}`);
  if (!exists) {
    throw new Error('Refresh token revoked');
  }

  // 3. 生成新 Access Token
  const newAccessToken = jwt.sign(
    {
      sub: payload.sub,
      email: payload.email,
      role: payload.role,
    },
    JMT_SECRET,
    { expiresIn: JMT_EXPIRES_IN },
  );

  return { accessToken: newAccessToken };
}

// 登出（撤销 Refresh Token）
export async function logout(userId: string, refreshToken: string) {
  await redis.del(`refresh_token:${userId}:${refreshToken}`);
}
```

### 3.3 密码安全

#### 3.3.1 密码存储（bcrypt）

```typescript
// 注册时：加密密码
import bcrypt from 'bcrypt';

async function register(email: string, password: string) {
  // 1. 验证密码强度
  if (!isStrongPassword(password)) {
    throw new Error('Password too weak');
  }

  // 2. 使用 bcrypt 加密密码（saltRounds = 12）
  const passwordHash = await bcrypt.hash(password, 12);

  // 3. 存储到数据库
  const user = await prisma.user.create({
    data: {
      email,
      passwordHash, // ❌ 不要存储明文密码！
    },
  });

  return user;
}

// 登录时：验证密码
async function login(email: string, password: string) {
  const user = await prisma.user.findUnique({ where: { email } });

  if (!user) {
    // ⚠️ 故意延迟响应（防止时序攻击）
    await bcrypt.compare(password, '$2b$12$fakehashfor timingattack');
    throw new Error('Invalid credentials');
  }

  // 比较密码
  const isValid = await bcrypt.compare(password, user.passwordHash);
  if (!isValid) {
    throw new Error('Invalid credentials');
  }

  return user;
}
```

#### 3.3.2 密码强度要求

```typescript
// 密码强度验证
function isStrongPassword(password: string): boolean {
  // 至少 8 位
  if (password.length < 8) return false;

  // 至少包含：大写字母、小写字母、数字、特殊字符
  const hasUpperCase = /[A-Z]/.test(password);
  const hasLowerCase = /[a-z]/.test(password);
  const hasNumber = /\d/.test(password);
  const hasSpecialChar = /[!@#$%^&*()_+\-=\[\]{};':"\\|,.<>\/?]/.test(password);

  return hasUpperCase && hasLowerCase && hasNumber && hasSpecialChar;
}
```

## 4. 授权设计

### 4.1 基于角色的访问控制（RBAC）

#### 4.1.1 角色定义

| 角色                    | 权限                                       |
| ----------------------- | ------------------------------------------ |
| **guest（游客）**       | 查看公开行程、搜索城市/景点                |
| **user（普通用户）**    | 创建/编辑/删除自己的行程、收藏景点         |
| **premium（付费用户）** | 普通用户权限 + 导出 PDF + 优先客服         |
| **admin（管理员）**     | 管理所有用户、管理城市知识库、查看审计日志 |

#### 4.1.2 权限检查中间件

```typescript
// apps/api/src/middlewares/auth.middleware.ts
import { FastifyRequest, FastifyReply } from 'fastify';

// 认证中间件（验证 JWT）
export async function authenticate(request: FastifyRequest, reply: FastifyReply) {
  try {
    // 1. 从请求头提取 Token
    const authHeader = request.headers.authorization;
    if (!authHeader?.startsWith('Bearer ')) {
      return reply.status(401).send({ error: 'Missing token' });
    }

    const token = authHeader.slice(7);

    // 2. 验证 Token
    const payload = verifyAccessToken(token);

    // 3. 将用户信息附加到请求对象
    request.user = {
      id: payload.sub,
      email: payload.email,
      role: payload.role,
    };
  } catch (error) {
    return reply.status(401).send({ error: 'Invalid or expired token' });
  }
}

// 授权中间件（检查角色）
export function authorize(allowedRoles: string[]) {
  return async (request: FastifyRequest, reply: FastifyReply) => {
    if (!request.user) {
      return reply.status(401).send({ error: 'Unauthorized' });
    }

    if (!allowedRoles.includes(request.user.role)) {
      return reply.status(403).send({ error: 'Forbidden' });
    }
  };
}
```

#### 4.1.3 使用中间件

```typescript
// 普通用户接口（需要登录）
app.get(
  '/api/v1/plans',
  {
    preHandler: [authenticate],
  },
  async (request, reply) => {
    const plans = await getPlansByUserId(request.user.id);
    return { data: plans };
  },
);

// 管理员接口（需要 admin 角色）
app.get(
  '/api/v1/admin/users',
  {
    preHandler: [authenticate, authorize(['admin'])],
  },

  },
);
```

### 5.2 存储层加密

#### 5.2.1 敏感字段加密（AES-256-GCM）

```typescript
// apps/api/src/utils/encryption.ts
import crypto from 'crypto';

const ALGORITHM = 'aes-256-gcm';
const SECRET_KEY = Buffer.from(process.env.ENCRYPTION_KEY!, 'hex'); // 32 字节密钥

// 加密
export function encrypt(plaintext: string): string {
  const iv = crypto.randomBytes(12); // 12 字节 IV（推荐 for GCM）
  const cipher = crypto.createCipheriv(ALGORITHM, SECRET_KEY, iv);

  const encrypted = Buffer.concat([cipher.update(plaintext, 'utf8'), cipher.final()]);

  const authTag = cipher.getAuthTag();

  // 返回格式：IV:AuthTag:EncryptedData（Base64 编码）
  return [iv.toString('base64'), authTag.toString('base64'), encrypted.toString('base64')].join(
    ':',
  );
}

// 解密
export function decrypt(ciphertext: string): string {
  const [ivBase64, authTagBase64, encryptedBase64] = ciphertext.split(':');

  const iv = Buffer.from(ivBase64, 'base64');
  const authTag = Buffer.from(authTagBase64, 'base64');
  const encrypted = Buffer.from(encryptedBase64, 'base64');

  const decipher = crypto.createDecipheriv(ALGORITHM, SECRET_KEY, iv);
  decipher.setAuthTag(authTag);

  const decrypted = Buffer.concat([decipher.update(encrypted), decipher.final()]);

  return decrypted.toString('utf8');
}
```

#### 5.2.2 需要加密的字段

| 字段            | 加密方式            | 说明                          |
| --------------- | ------------------- | ----------------------------- |
| **用户密码**    | bcrypt（单向哈希）  | 无法解密，只能验证            |
| **用户邮箱**    | AES-256-GCM（可选） | 如果要求 GDPR 合规            |
| **用户手机号**  | AES-256-GCM         | 敏感信息                      |
| **支付信息**    | ❌ 不存储           | 使用第三方支付（微信/支付宝） |
| **LLM API Key** | AES-256-GCM         | 环境变量中存储加密后的 Key    |

## 6. 输入验证

### 6.1 使用 Schema 验证（Fastify Schema）

```typescript
// apps/api/src/routes/plan.routes.ts
import { FastifyInstance } from 'fastify';

export default async function planRoutes(app: FastifyInstance) {
  // 创建行程（输入验证）
  app.post(
    '/api/v1/plans',
    {
      schema: {
        body: {
          type: 'object',
          required: ['title', 'cityId', 'startDate', 'endDate'],
          properties: {
            title: {
              type: 'string',
              minLength: 1,
              maxLength: 100,
              pattern: '^[\\w\\s]+$', // 只允许字母、数字、下划线、空格
            },
            cityId: {
              type: 'string',
              pattern: '^[a-z]+$', // 只允许小写字母
            },
            startDate: {
              type: 'string',
              format: 'date', // 自动验证日期格式（ISO 8601）
            },
            endDate: {
              type: 'string',
              format: 'date',
            },
            travelers: {
              type: 'object',
              properties: {
                adults: { type: 'integer', minimum: 1, maximum: 10 },
                children: { type: 'integer', minimum: 0, maximum: 10 },
                elders: { type: 'integer', minimum: 0, maximum: 10 },
              },
            },
          },
          // 自定义验证：结束日期必须晚于开始日期
          if: {
            properties: {
              startDate: { type: 'string' },
              endDate: { type: 'string' },
            },
          },
          then: {
            not: {
              required: ['startDate', 'endDate'],
              properties: {
                startDate: { type: 'string' },
                endDate: { type: 'string', const: { $data: '1/startDate' } },
              },
            },
          },
        },
        response: {
          201: {
            type: 'object',
            properties: {
              data: { type: 'object' },
            },
          },
        },
      },
    },
    async (request, reply) => {
      const { title, cityId, startDate, endDate } = request.body;

      // 业务验证（在 Schema 验证之后）
      if (new Date(endDate) <= new Date(startDate)) {
        return reply.status(400).send({ error: 'End date must be after start date' });
      }

      const plan = await createPlan(request.user.id, request.body);
      return reply.status(201).send({ data: plan });
    },
  );
}
```

```typescript
// ✅ 安全：使用 Prisma（参数化查询）
async function getAttractionsByCity(cityId: string) {

  const result = await prisma.$queryRaw(query);
```

## 8. 安全头配置

### 8.1 使用 Helmet（Fastify）

```typescript
// apps/api/src/server.ts
import helmet from '@fastify/helmet';

app.register(helmet, {
  // 1. 防止 XSS
  xssFilter: true,

  // 2. 防止点击劫持（Clickjacking）
  frameguard: {
    action: 'deny', // 禁止被 iframe 嵌入
  },

  // 3. 防止 MIME 类型嗅探
  noSniff: true,

  // 4. 强制 HTTPS（生产环境）
  hsts: {
    maxAge: 31536000, // 1 年
    includeSubDomains: true,
    preload: true,
  },

  // 5. 内容安全策略（CSP）
  contentSecurityPolicy: {
```

## 9. 日志与审计

### 9.1 审计日志记录

```typescript
// apps/api/src/services/audit.service.ts
import Redis from 'ioredis';

const redis = new Redis(process.env.REDIS_URL);

export async function logAuditEvent(event: {
  userId: string;
  action: string;
  resource: string;
  resourceId: string;
  ip: string;
  userAgent: string;
  timestamp?: Date;
}) {
  const auditLog = {
    ...event,
    timestamp: event.timestamp || new Date(),
  };

  // 存储到 Redis（后续异步写入数据库）
  await redis.lpush('audit_logs', JSON.stringify(auditLog));
}

// 使用审计日志
app.delete('/api/v1/plans/:planId', { preHandler: [authenticate] }, async (request, reply) => {
  const { planId } = request.params;

  // 记录审计日志
  await logAuditEvent({
    userId: request.user.id,
    action: 'DELETE',
    resource: 'Plan',
    resourceId: planId,
    ip: request.ip,
    userAgent: request.headers['user-agent'] || '',
  });

  // 执行删除
  await deletePlan(planId, request.user.id);

  return reply.status(204).send();
});
```
