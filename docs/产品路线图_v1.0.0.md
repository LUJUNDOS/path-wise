# PATH-WISE 产品路线图 v1.0.0

> **文档说明**：本文档定义 PATH-WISE 旅游攻略生成平台三阶段功能规划与里程碑目标。
> **更新日期**：2026-06-22
> **版本**：v1.0.0

---

## 愿景与北极星指标

**产品愿景**：让每个人都能在 3 分钟内拥有一份专业级旅行攻略。

**北极星指标**：每周活跃用户生成攻略数（WAU × 人均生成次数）

**三年目标**：

- Year 1：验证核心流程，达到 MAU 1 万
- Year 2：规模化增长，MAU 10 万
- Year 3：商业化成熟，MAU 50 万

## 阶段一：MVP（2026 Q3）

### 目标

> **核心目标**：完成端到端流程验证 —— 用户输入目的地，AI 生成可读的旅游攻略，大交通展示正确。

**成功标准**：

- 能生成 5+ 城市的完整攻略（北京/上海/成都/西安/长沙）
- 攻略生成时间 < 30 秒
- 大交通展示完整（高铁/普速火车/航班/大巴）
- 免责声明正确显示（橙色，`#FA8C16`）
- 无 P0 Bug

### 功能清单（P0 必做）

| #   | 功能模块    | 描述                                       | 负责方     |
| --- | ----------- | ------------------------------------------ | ---------- |
| M01 | 城市搜索    | 支持中文城市名搜索，显示热门目的地         | 前端+后端  |
| M02 | 行程配置    | 天数（1-14天）、人数、出行日期、大交通偏好 | 前端+后端  |
| M03 | AI 攻略生成 | 调用 LLM 生成每日行程，SSE 流式推送        | 后端（AI） |
| M04 | 大交通展示  | 高铁/普速火车/航班/大巴，含隔夜标识        | 前端       |
| M05 | 天气提示    | 目的地天气概况（和风天气 API）             | 后端       |
| M06 | 免责声明    | 大交通信息免责声明（橙色）                 | 前端       |
| M07 | 城市知识库  | 5 个城市静态数据（POI + 大交通信息）       | 数据       |

### 功能清单（P1 可选）

| #   | 功能模块     | 描述                      | 状态         |
| --- | ------------ | ------------------------- | ------------ |
| M08 | 长途大巴支持 | bus 类型大交通展示        | 已设计，可做 |
| M09 | 预算估算     | 简单费用估算（人均/总价） | 可延后       |
| M10 | 攻略分享链接 | 生成可分享的 URL          | 可延后       |

### 技术里程碑（MVP）

```
Week 1-2:  项目初始化
           ├── Monorepo 搭建（前端+后端）
           ├── PostgreSQL + Redis 本地环境
           └── Prisma Schema 建表

Week 3-4:  城市知识库
           ├── 录入 5 个城市静态数据
           ├── city/list 接口
           └── city/knowledge 接口

Week 5-6:  核心引擎
           ├── Trip Lifecycle 算法实现
           ├── LLM 路由器实现（DeepSeek 优先）
           └── POST /trips/generate 接口

Week 7-8:  前端页面
           ├── 城市搜索页
           ├── 行程配置页
           └── 攻略展示页（含大交通卡片）

Week 9:    集成测试
           ├── 端到端测试（北京→长沙）
           └── 大交通展示验证

Week 10:   MVP 验收
           └── 对照 MVP验收标准文档 逐项检查
```

### MVP 验收标准（概要）

详见《MVP验收标准文档\_v1.0.0.md》，核心验收点：

1. ✅ 5 个目标城市均可成功生成攻略
2. ✅ 生成时间 P95 < 30 秒
3. ✅ 大交通类型全部正确展示
4. ✅ 缓冲时间计算正确（flight=120, rail=90, bus=60）
5. ✅ 免责声明显示（橙色 `#FA8C16`）
6. ✅ 移动端 375px 布局正常
7. ✅ 无 P0 Bug

## 阶段二：正式版（2026 Q4）

### 目标

> **核心目标**：增加用户账号体系、社交分享功能，提升留存率和传播力。

**成功标准**：

- MAU ≥ 5,000
- 次日留存 ≥ 30%
- 攻略分享转化率 ≥ 10%

### 功能清单

| 优先级 | 功能           | 描述                                    |
| ------ | -------------- | --------------------------------------- |
| P0     | 用户注册/登录  | 手机号 + 验证码，支持微信授权           |
| P0     | 历史攻略保存   | 用户可保存、查看历史攻略                |
| P0     | 攻略分享       | 生成分享卡片（图文）、可分享至微信/微博 |
| P1     | 收藏 & 点赞    | 用户可收藏他人攻略                      |
| P1     | 用户评论       | 攻略底部评论区                          |
| P1     | 个性化推荐     | 基于历史行程推荐目的地                  |
| P2     | 城市知识库扩展 | 从 5 个城市扩展到 20+ 城市              |
| P2     | 攻略模板       | 提供「亲子游」「蜜月游」「穷游」等模板  |
| P2     | 多语言支持     | 简体中文（已有）+ 繁体中文              |

### 技术里程碑（正式版）

```
Q4 Month 1: 账号体系
            ├── 用户注册/登录 API
            ├── JWT 认证
            └── 微信 OAuth 接入

Q4 Month 2: 内容存储与分享
            ├── 攻略保存/读取 API
            ├── 分享卡片生成（Canvas/Puppeteer）
            └── 分享链接生成

Q4 Month 3: 社交功能
            ├── 收藏/点赞 API
            ├── 评论功能
            └── 城市知识库扩展（20 城市）
```

### 数据指标目标（阶段二结束）

| 指标              | 目标值 |
| ----------------- | ------ |
| MAU               | 5,000+ |
| 次日留存          | ≥ 30%  |
| 人均生成攻略数/月 | ≥ 2 次 |
| 分享率            | ≥ 10%  |

## 阶段三：增长版（2027 Q1–Q2）

### 目标

> **核心目标**：商业化变现 + 智能化升级，建立平台护城河。

**成功标准**：

- MAU ≥ 50,000
- GMV（旅游相关交易额）≥ 100 万/月
- 攻略准确率用户评分 ≥ 4.5/5

### 功能清单

| 优先级 | 功能         | 描述                                           |
| ------ | ------------ | ---------------------------------------------- |
| P0     | 实时价格接入 | 接入 12306/携程 实时票价，替换 mock 数据       |
| P0     | 智能行程调整 | 用户可对话式调整攻略（「第二天改为看博物馆」） |
| P0     | 达人攻略市场 | 优质攻略作者认证，付费攻略购买                 |
| P1     | 多城市行程   | 支持「北京→西安→成都」跨城行程                 |
| P1     | 自驾模式     | 自驾交通逻辑（≠ 大交通逻辑）                   |
| P1     | 小程序版本   | 微信小程序/支付宝小程序                        |
| P2     | 广告变现     | 景区/酒店/餐饮广告植入（原生广告）             |
| P2     | 行程预订跳转 | 一键跳转携程/飞猪/12306 订票                   |
| P2     | 企业版       | 企业团建出行专属功能                           |
| P2     | 海外城市     | 支持港澳台及东南亚城市                         |

### 技术里程碑（增长版）

```
Q1: 智能化升级
    ├── 对话式行程调整（多轮 LLM）
    ├── 实时价格 API 接入
    └── 多城市行程算法

Q2: 商业化变现
    ├── 达人认证体系
    ├── 付费攻略支付（微信支付）
    └── 广告平台接入（腾讯广告/字节）
```

## 竞品对比与差异化

| 维度       | 携程攻略           | 马蜂窝             | PATH-WISE                |
| ---------- | ------------------ | ------------------ | ------------------------ |
| 攻略来源   | UGC（用户生成）    | UGC（用户生成）    | **AI 生成（个性化）**    |
| 生成速度   | 手动编辑（小时级） | 手动编辑（小时级） | **3 分钟**               |
| 大交通集成 | 独立预订           | 独立预订           | **行程内嵌展示**         |
| 个性化程度 | 低（搜索结果）     | 低（搜索结果）     | **高（根据偏好定制）**   |
| 实时调整   | 不支持             | 不支持             | **对话式调整（阶段三）** |

**核心差异化**：AI 生成 × 端到端集成（大交通 + 天气 + 行程）× 个性化定制

## 风险与应对

| 风险                | 概率 | 影响 | 应对策略                       |
| ------------------- | ---- | ---- | ------------------------------ |
| LLM API 成本超预算  | 中   | 高   | 多模型路由降低成本；用户级限流 |
| 高德/12306 API 变更 | 低   | 中   | Adapter 层隔离；Mock 数据兜底  |
| 攻略质量不达预期    | 中   | 高   | Prompt 迭代；用户反馈机制      |
| 竞品快速跟进        | 中   | 中   | 加快功能迭代；构建数据护城河   |
| LLM 内容合规风险    | 低   | 高   | 输出过滤；敏感词检测           |
