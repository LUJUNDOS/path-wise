## 一、问题诊断

对 `apps/web/` 进行全面审查后，识别出 6 个层次的问题：

| #   | 问题                                                           | 影响                             | 严重度 |
| --- | -------------------------------------------------------------- | -------------------------------- | ------ |
| 1   | **色彩系统无品牌个性** — shadcn 默认灰蓝配色，与旅行场景不匹配 | 用户第一印象冷淡，无情感连接     | P0     |
| 2   | **无暗黑模式** — CSS 仅 `:root`，无 `.dark` 覆盖               | 夜间使用刺眼，缺失现代标配       | P0     |
| 3   | **无自定义字体** — 全系统默认字体，排版层级单一                | 界面缺乏品质感和视觉节奏         | P0     |
| 4   | **表面无深度分层** — 所有卡片/按钮同一视觉平面                 | 信息层级模糊，用户注意力无处安放 | P1     |
| 5   | **生成中页面体验枯燥** — 仅旋转图标 + 虚线占位框               | 30-60s 等待无品牌感知，流失风险  | P1     |
| 6   | **品牌标识空白** — 无 Logo、无品牌图形语言                     | 用户记不住产品，无差异化         | P2     |

## 二、设计方向

**关键词**：探索 · 温暖 · 信任 · 精致

**设计目标**：从"后台管理工具"的视觉感受转变为"旅行灵感平台"。

**设计原则**：

1. **暖色调为主** — 用琥珀/金色传递旅行期待感
2. **深度分层** — 背景 → 表面 → 悬浮，三层清晰
3. **克制的装饰** — 渐变和阴影作为功能元素，不为装饰而装饰
4. **暗黑模式一等公民** — 所有新样式同步支持 dark mode

## 四、字体系统

### 4.1 字体选择

| 层级        | 字体                   | 来源                  | 用途                      |
| ----------- | ---------------------- | --------------------- | ------------------------- |
| **Display** | DM Serif Display       | Google Fonts          | 品牌名、Hero 标题（英文） |
| **Body**    | Inter + 中文系统字体栈 | Google Fonts + System | 正文、表单、组件          |

### 4.2 字体栈

```css
--font-display: 'DM Serif Display', 'Noto Serif SC', 'STSong', serif;
--font-sans: 'Inter', 'PingFang SC', 'Microsoft YaHei', 'Hiragino Sans GB', system-ui, sans-serif;
```

### 4.3 排版层级

| 层级      | 字号                    | 字重            | 行高              | 用途           |
| --------- | ----------------------- | --------------- | ----------------- | -------------- |
| Hero 标题 | `text-4xl` ~ `text-5xl` | `font-bold`     | `leading-tight`   | 首页主标题     |
| 页面标题  | `text-2xl` ~ `text-3xl` | `font-bold`     | `leading-tight`   | 各页 h1        |
| 区块标题  | `text-lg` ~ `text-xl`   | `font-semibold` | `leading-snug`    | Section header |
| 卡片标题  | `text-base`             | `font-semibold` | `leading-snug`    | CardTitle      |
| 正文      | `text-sm`               | `font-normal`   | `leading-relaxed` | 描述文字       |
| 辅助      | `text-xs`               | `font-normal`   | `leading-normal`  | 标签、提示     |

## 五、组件深度分层

### 5.1 Shadow 层级

| 层级    | Shadow 类     | 使用场景               |
| ------- | ------------- | ---------------------- |
| L0 平面 | `shadow-none` | 输入框、分隔线         |
| L1 微浮 | `shadow-sm`   | 卡片、面板             |
| L2 悬浮 | `shadow-md`   | Hover 卡片、下拉菜单   |
| L3 弹层 | `shadow-lg`   | Dialog、Popover、Modal |
| L4 顶层 | `shadow-xl`   | Toast、Tooltip         |

### 5.2 圆角层级

| 层级  | 值             | 使用场景            |
| ----- | -------------- | ------------------- |
| Sharp | `rounded-md`   | 输入框、按钮、Badge |
| Soft  | `rounded-lg`   | 卡片                |
| Round | `rounded-xl`   | 大面板、Hero 卡片   |
| Pill  | `rounded-full` | 标签、头像、Chip    |

## 七、验收标准

### P0 验收

- [ ] 页面主色调为暖琥珀/金色系，不再是灰蓝
- [ ] 暗黑模式可通过 `.dark` 类切换，色值正确无刺眼
- [ ] Google Fonts 加载成功，DM Serif Display 和 Inter 生效
- [ ] 所有现有组件在 light/dark 模式均正常显示
- [ ] 无控制台错误或样式丢失

### P1 验收

- [ ] 首页有 Hero 视觉区域
- [ ] 生成中页面使用骨架屏动画
- [ ] 卡片之间有清晰的深度层次
- [ ] 微交互流畅（hover、active、transition）

### P2 验收

- [ ] 补充组件可用且风格一致
- [ ] TripResultPage 信息架构合理
- [ ] 所有 shadcn 组件与新色系协调

---

## 八、技术约束

1. **不破坏 shadcn/ui** — CSS 变量名保持 shadcn 约定，仅改值
2. **不引入新依赖** — 字体用 Google Fonts CDN（非 npm 包），无需 bundle
3. **向后兼容** — 现有 Tailwind 类名全部有效，仅增强
4. **渐进式** — 先改全局（P0），再逐个页面升级（P1），最后补细节（P2）
