## 🔑 前端交互设计必须知道的核心决策

### 1. SSE流式返回（不能用传统HTTP）

生成行程耗时约30-90s，必须流式返回，否则用户等待超时。

**SSE事件流顺序**：

```
connected → progress → day_ready（每天生成完立即推送）→ done
```

- `day_ready` 携带当天完整行程JSON，前端到了就渲染
- `accommodation` 合并到每个城市第一天的 `day_ready` 事件
- 前端必须做**渐进渲染**，不用等全部完成
- 详细格式见 `API接口设计规格书_v1.0.0.md` 第4章

### 2. 用户输入冲突检测（6条规则，均提示不阻断）

| #   | 冲突条件                                                                  | 前端提示方式                                                           |
| --- | ------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| 1   | `budget=穷游` + `accommodation=精品/豪华`                                 | Toast + 提示："穷游预算下精品/豪华住宿可能超支，建议调整"              |
| 2   | `budget=穷游` + `dining` 含"高端餐厅"                                     | Toast + 提示："穷游预算下高端餐饮可能超支"                             |
| 3   | `travelers.elders` 非空 + `pace=高强度`                                   | 弹窗提示："当前有老人同行，高强度节奏可能不适合，建议调整为舒适或悠闲" |
| 4   | `travelers.children` 有 `<3岁` + `destinations[].transportTo` 含`高铁>4h` | 弹窗提示："有低龄幼儿同行，长途高铁建议拆分为短途或改乘飞机"           |
| 5   | `pace=高强度` + `travelers.elders` 非空或 `children` 非空                 | 弹窗提示："高强度节奏+老幼同行，建议调整为舒适"                        |
| 6   | `budget=豪华` + `accommodation=青旅/民宿`                                 | Toast提示："豪华预算下青旅体验落差较大，确认坚持此选择？"              |

- 所有提示均为**非阻断**，用户可点击"坚持原选择"关闭
- 前端弹窗组件：`ConflictWarningModal`

### 3. 饮食偏好（3个选项，非输入框）

```
[ ] 本地特色（推荐当地菜）
[ ] 连锁餐厅（稳妥选择）
[ ] 网红店（打卡热门餐厅）
```

- 去掉了清真/素食/不吃辣/海鲜过敏（到店提要求更合适）
- 对应 `preferences.dining: string[]`

### 4. 大交通信息展示（MVP阶段不做实时API）

- 数据来源：静态知识库JSON的 `transportOptions` 字段（经验推荐）
- 前端**必须标注**：`⚠️ 车次信息仅供参考，请尽快到12306/携程/飞猪订票`
- 不提供订票跳转（MVP阶段）
- 阶段二可考虑接入第三方聚合API（¥0.01~0.05/次）

### 5. MVP阶段无LLM响应缓存（⚠️ 重要变更）

**决策**：MVP阶段**去掉LLM响应缓存**，每次行程生成都直接调用LLM API。

**原因**：

- cache_key字段越多命中率越低（用户几乎每次请求都是不同组合）
- DeepSeek-V3成本低（约¥0.1~0.5/次），成本压力不大
- 避免返回过期/不精准的缓存内容（季节变化导致攻略失效）
- 简化系统复杂度，聚焦核心功能

**对前端的影响**：

- 不需要传递特殊字段用于缓存key计算
- 每次生成都是全新请求，不存在"缓存命中"状态
- API响应中无 `cacheHit` 字段

**阶段二考虑**：待有真实用户数据后，再决定是否 reintroduce 缓存（基于数据分析）

### 6. LLM模型选择（前端展示用，用户不可选）

- 平台统一提供API Key（环境变量），用户无感知
- 模型优先级：DeepSeek-V3(P0) → 智谱GLM-4(P1) → 小米MiLM(P2) → Kimi-K2(P3)
- 前端可展示"当前使用：DeepSeek-V3"字样（可选，不暴露降级）
