## 1. 概述

### 1.1 文档目的

本文档定义 PATH-WISE 的数据分析体系，包括：

- 核心业务指标（北极星指标）
- 埋点事件规范
- 用户行为漏斗分析
- 数据看板设计
- 技术实现方案

### 1.2 数据收集原则

| 原则             | 说明                                 |
| ---------------- | ------------------------------------ |
| **最小必要原则** | 只收集分析必要的数据，不过度收集     |
| **用户知情同意** | 在隐私政策中明确说明数据收集行为     |
| **数据匿名化**   | 分析数据时对用户 ID 进行哈希处理     |
| **数据安全**     | 埋点数据传输使用 HTTPS，不含敏感信息 |

### 1.3 工具选型

| 工具                   | 用途                     | 推荐                              |
| ---------------------- | ------------------------ | --------------------------------- |
| **Mixpanel**           | 用户行为分析、漏斗分析   | ✅ 推荐（功能全面，免费层够用）   |
| **PostHog**            | 开源替代方案（可自托管） | ✅ 备选（数据主权更好）           |
| **Google Analytics 4** | 页面 PV/UV 统计          | ⚠️ 国内访问慢，不推荐             |
| **阿里云 DataV**       | 自定义数据看板           | ✅ 推荐（国内访问，与阿里云集成） |
| **Grafana**            | 技术指标看板             | ✅ 推荐（与 Prometheus 集成）     |

## 2. 核心指标体系

### 2.1 北极星指标（North Star Metric）

**PATH-WISE 的北极星指标**：

> **用户完成行程并满意使用** = 月活跃用户中，成功生成攻略且评分 ≥ 4 分的用户数

**选择理由**：

- 代表用户获得了核心价值（有了一份满意的旅游攻略）
- 可拆解为多个子指标（访问 → 配置 → 生成 → 满意）

### 2.2 指标层级体系

```
北极星指标：月有效行程数（成功生成 + 评分 ≥4 分）
    │
    ├── L1 增长指标
    │   ├── 月活跃用户数（MAU）
    │   ├── 新用户注册转化率
    │   └── 用户留存率（次日/7日/30日）
    │
    ├── L2 参与指标
    │   ├── 行程生成次数
    │   ├── 行程完成率（配置完成 → 生成成功）
    │   ├── 平均每用户行程数
    │   └── 用户满意度（NPS / 评分）
    │
    ├── L3 功能指标
    │   ├── 各城市行程生成分布
    │   ├── 交通类型分布（高铁/飞机/普速火车/大巴）
    │   ├── 平均行程天数
    │   ├── LLM 模型使用分布
    │   └── 平均生成耗时
    │
    └── L4 技术指标
        ├── API 响应时间（P50/P95/P99）
        ├── 页面加载时间（FCP/LCP）
        ├── LLM 调用成功率
        └── 错误率
```

### 2.3 关键指标定义与计算

| 指标名称           | 定义                                    | 计算方式                      | 目标值（MVP 阶段） |
| ------------------ | --------------------------------------- | ----------------------------- | ------------------ |
| **行程生成完成率** | 成功生成的行程 / 点击"生成行程"按钮次数 | `completed / initiated`       | ≥ 85%              |
| **配置完成率**     | 完成所有配置步骤 / 进入配置页次数       | `configured / entered_config` | ≥ 70%              |
| **次日留存率**     | 次日回访 / 首日注册                     | `DAU(T+1) / new_register(T)`  | ≥ 30%              |
| **7日留存率**      | 7天内回访 / 首日注册                    | -                             | ≥ 20%              |
| **平均满意度评分** | 用户对生成结果的评分                    | `sum(score) / count`          | ≥ 4.0/5.0          |
| **LLM 成功率**     | 成功生成 / 总调用次数                   | -                             | ≥ 95%              |

## 3. 埋点规范

### 3.1 事件命名规范

```
命名格式：<页面>_<对象>_<动作>

规则：
  - 全小写，用下划线连接
  - 页面：city_select / trip_config / plan_view / home / auth
  - 对象：city_card / generate_btn / transport_option / day_card 等
  - 动作：click / view / select / submit / complete / error 等

示例：
  city_select_city_card_click      → 城市选择页点击城市卡片
  trip_config_generate_btn_click   → 行程配置页点击生成按钮
  plan_view_day_card_expand        → 行程详情页展开单日卡片
  auth_register_form_submit        → 注册表单提交
```

### 3.2 公共属性（每个事件自动附加）

```typescript
// 每条埋点事件都自动附加以下属性

interface CommonEventProperties {
  // 用户信息
  user_id: string | null; // 用户 ID（匿名用 session_id）
  session_id: string; // 会话 ID（同一次访问共享）
  is_logged_in: boolean;

  // 设备信息
  platform: 'web_mobile' | 'web_desktop' | 'web_tablet';
  os: string; // 操作系统（如 "iOS 17.0"）
```

```typescript
// apps/web/src/hooks/useAnalytics.ts
import Mixpanel from 'mixpanel-browser';

export function useAnalytics() {
  const track = useCallback((
    eventName: string,
    properties?: Record<string, unknown>
  ) => {
    // 只在生产环境发送
    if (process.env.NODE_ENV !== 'production') {
      console.debug('[Analytics]', eventName, properties);
      return;
    }

    Mixpanel.track(eventName, {
      ...properties,
      timestamp: new Date().toISOString(),
    });
  }, []);

  return { track };
}

// 使用示例
function CityCard({ city }: { city: City }) {

      card_position: position,  // 在列表中的位置（1-N）
    });
    navigate(`/config?city=${city.id}`);
  };

  return <div onClick={handleClick}>...</div>;
}
```

## 4. 用户行为漏斗

### 4.1 核心转化漏斗

```
用户访问首页
    │ 转化率目标：70%
    ▼
进入城市选择页
    │ 转化率目标：80%
    ▼
选择目的地城市
    │ 转化率目标：90%
    ▼
进入行程配置页
    │ 转化率目标：75%
    ▼
完成行程配置（填写所有必填项）
    │ 转化率目标：85%
    ▼
点击"生成行程"按钮
    │ 转化率目标：95%（技术成功率）
    ▼
行程生成成功
    │ 转化率目标：80%
    ▼
用户查看完整行程（停留 > 30 秒）
    │ 转化率目标：70%
    ▼
用户满意（评分 ≥ 4 分 / 保存行程）
```

**整体漏斗转化率**：访问首页 → 满意完成 ≈ 25-35%

### 4.2 配置页子漏斗（排查流失原因）

```
进入配置页
    │
    ├── 选择出发城市
    ├── 选择目的地（此处可能流失：城市不在支持列表）
    ├── 选择日期范围（此处可能流失：日期选择交互复杂）
    ├── 填写人数（此处流失少）
    ├── 选择交通方式（此处可能流失：不理解选项含义）
    └── 点击生成（此处可能流失：用户犹豫 / 不信任 AI）
```

## 7. 用户分群

### 7.1 用户分层

| 分层         | 定义                     | 占比估算 | 运营策略                     |
| ------------ | ------------------------ | -------- | ---------------------------- |
| **超级用户** | 30天内生成 ≥ 5 次行程    | 5%       | 邀请内测、征集反馈、付费转化 |
| **活跃用户** | 30天内生成 1-4 次行程    | 25%      | 功能引导、促活推送           |
| **沉睡用户** | 注册后 7-30 天未生成行程 | 30%      | 激活推送（展示热门行程示例） |
| **流失用户** | 注册后 30 天以上未登录   | 40%      | 召回推送（限时优惠）         |

## 8. 技术实现

### 8.1 埋点 SDK 集成（前端）

```typescript
// apps/web/src/lib/analytics.ts

import Mixpanel from 'mixpanel-browser';

// 初始化
export function initAnalytics() {
  if (process.env.NODE_ENV !== 'production') return;

  Mixpanel.init(process.env.VITE_MIXPANEL_TOKEN!, {
    debug: false,
    track_pageview: true, // 自动追踪页面访问
    persistence: 'localStorage',
    ip: false, // 不追踪 IP（隐私保护）
  });
}

// 标识用户（登录后调用）
export function identifyUser(userId: string, profile: UserProfile) {
  Mixpanel.identify(userId);
  Mixpanel.people.set({
    $name: profile.nickname,
    $email: profile.email, // 注意：不展示在 Mixpanel UI 中，仅用于匹配
    role: profile.role,
    created_at: profile.createdAt,
    plan_count: profile.planCount,
  });
}

// 重置用户（退出登录后调用）
export function resetUser() {
  Mixpanel.reset();
}

// 统一追踪函数
export function track(eventName: string, properties?: Record<string, unknown>) {
  if (process.env.NODE_ENV !== 'production') {
    console.debug('[Analytics Track]', eventName, properties);
    return;
  }
  Mixpanel.track(eventName, properties);
}
```

### 8.2 服务端事件（后端）

```typescript
// apps/api/src/services/analytics.service.ts
// 服务端埋点用于追踪后端关键事件（LLM 调用、生成成功/失败）

import Mixpanel from 'mixpanel';

const mixpanel = Mixpanel.init(process.env.MIXPANEL_TOKEN!);

export function trackServerEvent(
  eventName: string,
  userId: string,
  properties: Record<string, unknown>,
) {
  if (process.env.NODE_ENV !== 'production') return;

  mixpanel.track(eventName, {
    distinct_id: userId,
    ...properties,
    $insert_id: generateUUID(), // 防止重复计数
    time: Date.now() / 1000,
  });
}

// 使用示例：行程生成完成
export async function onPlanGenerationCompleted(
  planId: string,
  userId: string,
  durationMs: number,
  llmModel: string,
) {
  trackServerEvent('plan_generation_completed', userId, {
    plan_id: planId,
    duration_ms: durationMs,
    llm_model: llmModel,
    source: 'server',
  });
}
```

### 8.3 数据隐私合规

```typescript
// 隐私合规处理

// 1. 用户同意前不发送任何埋点数据
let analyticsConsented = false;

export function setAnalyticsConsent(consented: boolean) {
  analyticsConsented = consented;
  localStorage.setItem('analytics_consent', String(consented));

  if (consented) {
    initAnalytics();
  } else {
    Mixpanel.opt_out_tracking(); // 停止追踪
  }
}

// 2. 数据脱敏：行程标题、城市名等正常记录，用户真实姓名不记录
function sanitizeProperties(properties: Record<string, unknown>) {
  const SENSITIVE_KEYS = ['name', 'phone', 'address', 'id_number'];

  return Object.fromEntries(
    Object.entries(properties).filter(([key]) => !SENSITIVE_KEYS.includes(key)),
  );
}

// 3. 用户删除账号时删除 Mixpanel 数据
export async function deleteUserAnalyticsData(userId: string) {
  // 发送删除请求到 Mixpanel GDPR API
  await fetch('https://mixpanel.com/api/app/data-deletions/v3.0/', {
    method: 'POST',
    headers: {
      Authorization: `Basic ${btoa(process.env.MIXPANEL_SERVICE_ACCOUNT!)}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      distinct_ids: [userId],
      compliance_type: 'CCPA',
    }),
  });
}
```
